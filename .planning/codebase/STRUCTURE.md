## Directory Purposes

**`backend/`:**
- Purpose: Python FastAPI REST API and browser orchestration
- Contains: Route handlers, models, database operations, VNC management, browser lifecycle
- Key files: `main.py` (app logic), `browser_manager.py` (Chromium management), `database.py` (persistence)

**`backend/tests/`:**
- Purpose: Unit test suite (pytest)
- Contains: Test cases for API routes, authentication, RFB filtering, database operations
- Run with: `pytest backend/tests/` (from project root)

**`frontend/src/`:**
- Purpose: React dashboard source code
- Contains: Components for profile management, VNC viewer, authentication
- Entry: `main.tsx` renders `App.tsx` into `#root` element

**`frontend/src/components/`:**
- Purpose: React components
- Key files:
  - `ProfileList.tsx` - Lists all profiles with status badges
  - `ProfileForm.tsx` - Create/edit profile form (21KB, complex form with many fields)
  - `ProfileViewer.tsx` - noVNC integration, VNC WebSocket connection, clipboard sync
  - `LoginPage.tsx` - Auth token input form
  - `LaunchButton.tsx` - Start/stop button with loading state
  - `StatusIndicator.tsx` - Status badge renderer

**`frontend/src/hooks/`:**
- Purpose: Custom React hooks
- Key files:
  - `useProfiles.ts` - Profile CRUD hook with 3-second polling refresh
  - `useProfiles.test.ts` - Hook unit tests (MSW mock API)

**`frontend/src/lib/`:**
- Purpose: Utility modules
- Key files:
  - `api.ts` - Fetch-based HTTP client with types for all endpoints
  - `api.test.ts` - Mock fetch tests for API client

**`frontend/src/styles/`:**
- Purpose: Global CSS (Tailwind)
- Contains: `globals.css` with custom color variables and component utilities

**`frontend/dist/`:**
- Purpose: Production build output (NOT committed, generated by `npm run build`)
- Generated: Vite bundles React + CSS + JS into optimized chunks
- Served by: FastAPI's StaticFiles at `/`

## Key File Locations

**Entry Points:**

**Backend:**
- `backend/main.py` - FastAPI app creation, middleware setup, route definitions, lifespan context
  - Line ~382: `app = FastAPI(...)`
  - Line ~373-380: Lifespan context manager (db init, cleanup on shutdown)

**Frontend:**
- `frontend/src/main.tsx` - React DOM render
  - Mounts App component into #root element

  - State: authState (checking/required/ok/error), selectedProfile, view (empty/create/edit/view)
  - Child: AppContent (main dashboard layout)

**Configuration:**

- `backend/requirements.txt` - Python dependencies (fastapi, uvicorn, pydantic, cloakbrowser, websockets, httpx)

- `backend/main.py` lines ~42-46 - Logging configuration (INFO level, format with timestamp)
- `backend/main.py` lines ~48-54 - Authentication configuration (AUTH_TOKEN env var)
- `frontend/src/lib/api.ts` - API base paths (all routes under `/api/`)

**Core Logic:**

## Naming Conventions

**Files:**

**Python:**
- Modules: `snake_case.py` (e.g., `browser_manager.py`, `vnc_manager.py`)
- Tests: `test_*.py` (e.g., `test_api.py`, `test_rfb.py`)

**TypeScript/React:**
- Components: `PascalCase.tsx` (e.g., `ProfileList.tsx`, `LaunchButton.tsx`)
- Hooks: `useCamelCase.ts` (e.g., `useProfiles.ts`)
- Utilities: `camelCase.ts` (e.g., `api.ts`)
- Tests: `*.test.ts` or `*.test.tsx` (e.g., `useProfiles.test.ts`)

**Directories:**
- All lowercase: `backend/`, `frontend/`, `components/`, `hooks/`, `lib/`, `tests/`, `styles/`

**Functions:**

**Python:**
- Public: `snake_case` (e.g., `list_profiles()`, `launch_persistent_context_async()`)
- Private: `_snake_case` (e.g., `_normalize_proxy()`, `_filter_rfb_client_messages()`)

**TypeScript/React:**
- Functions: `camelCase` (e.g., `handleSelect()`, `setOnUnauthorized()`)
- Event handlers: `handle{Action}` (e.g., `handleCreate()`, `handleLaunch()`)
- Getters: `get{Thing}` (e.g., `getProfile()`, `getStatus()`)

**Variables:**

**Python:**
- Constants: `UPPER_SNAKE_CASE` (e.g., `AUTH_TOKEN`, `BASE_DISPLAY`, `_AUTH_EXEMPT`)
- State dicts: `snake_case` (e.g., `running`, `_allocated`)

**TypeScript/React:**
- State: `camelCase` (e.g., `profiles`, `selectedId`, `authState`)
- Props: `camelCase` (e.g., `profileId`, `onDisconnect`, `clipboardSync`)
- Types: `PascalCase` (e.g., `Profile`, `ProfileCreateData`, `AuthState`)

## Where to Add New Code

**New API Endpoint:**
1. Define Pydantic request/response models in `backend/models.py`
2. Add route handler in `backend/main.py`
   - Use `@app.get()`, `@app.post()`, etc. decorators
   - Validate auth via AuthMiddleware (automatic unless in _AUTH_EXEMPT)
3. Add TypeScript types to `frontend/src/lib/api.ts` (interfaces + api.* function)
4. Add React component or hook to call the API (in `frontend/src/components/` or `frontend/src/hooks/`)
5. Write tests: `backend/tests/test_api.py` and/or `frontend/src/lib/api.test.ts`

**New React Component:**
1. Create file in `frontend/src/components/{ComponentName}.tsx`
2. Follow naming: `export function ComponentName() { ... }`
3. Use Tailwind classes for styling (see `globals.css` for color variables)
4. Import at call site (e.g., in `App.tsx` or another component)
5. Add tests alongside: `frontend/src/components/{ComponentName}.test.tsx` (vitest + @testing-library/react)

**New Hook:**
1. Create file in `frontend/src/hooks/use{HookName}.ts`
2. Return object with state and setter functions
3. Example pattern: See `useProfiles.ts` (state + action functions)
4. Add tests: `frontend/src/hooks/use{HookName}.test.ts`

**New Database Table:**
1. Add schema to `backend/database.py` init_db() function (SQLite CREATE TABLE)
2. Add migration check (see line ~70-76 for example: `if "column" not in cols`)
3. Add CRUD functions below schema (`get_`, `list_`, `create_`, `update_`, `delete_`)
4. Add tests in `backend/tests/test_database.py`

**New External Integration:**
1. Add dependency to `backend/requirements.txt` or `frontend/package.json`
2. Create module file (e.g., `backend/integrations/{service}.py`)
3. Wrap SDK client and error handling
4. Call from business logic (e.g., BrowserManager, API handlers)
5. Add tests with mocked SDK

## Special Directories

**`backend/tests/`:**
- Purpose: Unit tests (pytest)
- Generated: No
- Committed: Yes
- Config: `pytest.ini` or pyproject.toml `[tool.pytest.ini_options]`
- Fixtures: `conftest.py` provides MockBrowserManager, MockVNC, temp database

**`frontend/dist/`:**
- Purpose: Production build output
- Generated: Yes (by `npm run build`)
- Committed: No (in .gitignore)
- Served by: FastAPI's StaticFiles middleware (line ~180 in main.py)

**`.planning/codebase/`:**
- Purpose: GSD codebase analysis documents
- Generated: No (manual creation by GSD tools)
- Committed: Yes (for future reference)
- Contains: ARCHITECTURE.md, STRUCTURE.md, CONVENTIONS.md, TESTING.md, etc.

**`/data/` (runtime):**
- Purpose: Persistent data directory (mounted as volume in Docker)
- Location: `/data/profiles/` - Profile user data directories
- Location: `/data/profiles.db` - SQLite database file
- Created by: Database init on app startup
- Committed: No (runtime data)
