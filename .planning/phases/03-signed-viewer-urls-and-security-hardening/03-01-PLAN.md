  <name>viewer_tokens module</name>

  <behavior>
    - mint("prof-1") returns str; decoded payload has profile_id=="prof-1", jti len>=32, exp > iat
    - validate(token, "prof-1") returns dict with same profile_id
    - validate(token, "other") raises ViewerTokenError
    - Expired token (monkeypatch time or ttl=1 sleep) raises ViewerTokenError
    - Tampered token raises ViewerTokenError
    - consume_jti(jti) first call True; second call False
    - After exp passes, evict_jtis removes stale jti entries (test with injected clock or short TTL)
  </behavior>
  <implementation>
    - class ViewerTokenError(Exception)
    - Use PyJWT; claim key `profile_id` (not `sub`)
    - jti = uuid4().hex
    - _pending_jtis: dict[str, float] mapping jti -> exp timestamp; consume marks used set or removes from pending
    - Read VIEWER_SECRET from os.environ; read VIEWER_TOKEN_TTL_SECS default 300
    - DEV_MODE: allow test secret from conftest monkeypatch
  </implementation>
</feature>
