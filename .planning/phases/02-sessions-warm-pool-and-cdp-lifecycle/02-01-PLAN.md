**Part A — Refactor `stop()` in `backend/browser_manager.py`.**

Replace the existing `stop()` method (around line 299):

```python
    async def stop(self, profile_id: str):
        """Stop a running browser instance."""
        # Pop before close so _on_browser_closed() finds nothing to clean up
        async with self._lock:
            running = self.running.pop(profile_id, None)

        if not running:
            return

        logger.info("Stopping profile %s", profile_id)

        try:
            await running.context.close()
        except Exception as exc:
            logger.warning("Error closing context for %s: %s", profile_id, exc)

        await self.vnc.stop_vnc(running.display)
```

with this two-method form (preserves exact external semantics):

```python
    async def stop(self, profile_id: str) -> None:
        """Stop a running browser instance.

        Public API: acquires self._lock then calls _stop_locked(). External
        callers (main.py /stop endpoint, /delete endpoint) use this.
        """
        async with self._lock:
            await self._stop_locked(profile_id)

    async def _stop_locked(self, profile_id: str) -> None:
        """Inner stop helper — caller MUST already hold self._lock.

        Used by SessionManager idle task (Plan 02-04), which wants to
        re-check attach counts under _lock and then stop without releasing.
        Calling stop() from inside _lock would deadlock — asyncio.Lock is
        not re-entrant. RESEARCH.md §"Pitfall C".
        """
        running = self.running.pop(profile_id, None)
        if not running:
            return
        logger.info("Stopping profile %s", profile_id)
        try:
            await running.context.close()
        except Exception as exc:
            logger.warning("Error closing context for %s: %s", profile_id, exc)
        await self.vnc.stop_vnc(running.display)
```

External call sites (`main.py` lines 537, 587 and the upcoming `routers/profiles.py`) continue to use `stop()` — no changes needed there.
