## Cross-phase integration (gsd-integration-checker)

### Passed (20)

Machine `POST /sessions` → upsert → launch → `cdp_url` + `vnc_viewer_url#token=`; viewer JWT/JTI/replay; `viewer_attach_count` on signed viewer WS; CDP WS API-key + attach counts; admin sessions list API + UI; template admin vs machine auth segregation; legacy launch/CRUD → 410; production env fail-closed; CSP headers.

### Wiring gaps (4) — tech debt, not missing Main App features

1. **Admin VNC vs warm pool** — `vnc_proxy` in `backend/main.py` does not call `SessionManager.on_attach` or bump `viewer_attach_count` (unlike `backend/routers/viewer.py`). Risk: idle shutdown while operator watches in admin UI.
2. **Admin clipboard poll** — `ProfileViewer` polls `api.getClipboard` → admin route that requires viewer token (403). Viewer-scoped read exists at `profiles_clipboard.py` but admin UI does not use it.
3. **Idle session UX** — `App.tsx` only opens `ProfileViewer` when `state === "running"`; `idle` profiles show stopped messaging despite possible running Chromium.
4. **Orphan API client** — `api.deleteProfile` still targets `DELETE /api/profiles/{id}` (kept intentionally for ops cleanup; not 410).
